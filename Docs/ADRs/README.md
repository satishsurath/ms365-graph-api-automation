# ADRs

This folder holds architecture decision records for the repo.

## Conventions

- Use sequential filenames such as `0001-topic-name.md`.
- Keep ADRs short, decision-oriented, and explicit about status.
- Typical statuses: `Proposed`, `Accepted`, `Superseded`, `Deprecated`.
- Update the relevant docs when an ADR changes user-facing behavior or implementation direction.
